Preguntas LAB 1

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet es una red de computadores que se encuentran conectadas alrededor de todo el mundo, la World Wide Web es una coleccion de paginas web a las que se accede conectándose a esta red de computadoras. Es decir, la diferencia es que el internet no necesita d ela web para funcionar pero la web si.

  • ¿Cuáles son las partes de un URL?

    Una url se conforma de un protocolo, dominio, path y parametros.
    El protocolo es el sistema de reglas que permiten la comunicacion en el internet, el dominio es el nombre único que identifica a un sitio web, el path es la ruta o carpeta del sitio web a la que deseamos acceder y los parametros son informacion especial que se añade a la liga para especificar los parametros que recibira el servidor que reciba nuestra peticion en la ruta especificada.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: método que se utiliza para pedir un recurso en específico
    • HEAD: método que solicita los encabezados de pedir cierto recurso
    • POST: método que se utiiza para enviar datos en el cuerpo del request
    • PUT: método que se utiliza para reemplazar todas las instancias de un recurso con la carga actual
    • PATCH: método que se utiliza para actualizar datos almacenados en la nube
    • DELETE: método que se utiliza para eliminar datos en la nube
  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método que se debe utilizar es el de post, ya que este metodo se utiliza para publicar informacion en un sitio. Esto depende de la forma en que se programa la forma.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    El método de HTTP que e usa en este caso es el Get porque se esta solicitando la informacion a una url que va a responder con datos en formato de HTML

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Las respuesta que inician con 200 indican que la peticion fue exitosa y se realizó una operación con éxito por lo que no sucedió ningún error

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    No, porque el error 404 indica que el recurso al que se intenta acceder no existe. Por ende, es el usuario el que debe cambiar su petición

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Si, esto se debe a que un error con codigo 500 sucede por un eror interno en el servidor, es decir, un error en la programación que es respondabilidad de los desarrolladores

    La nueva version además de incluir nuevos elementos semánticos, ahora permite incluir elementos multimedia sin necesidad de otras herramientas como Flash y expande el repertorio de tipos de entradas en los formularios. Además HTML5, promueve separar los componentes de esttilo y funcionalidad y ponerlos en otros archivos

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    Una tabla que creada usando HTML se compone de un elemento table que de forma similar a una lista tiene subelementos, en este caso se usan elementos tr para las filas y dentro de estos se ponene elementos td para cada celda, Alternativamente, en CSS esta estructura se puede crear usando una forma de layout onocida como grid que es especialmente útil para la creacion de layouts responsivos

  • Sobre el ciclo de vida y desarrollo de los sistemas de información:
    • ¿Cuál es el ciclo de vida de los sistemas de información?

      El ciclo de vida de estos sistema consta de 5 fases:
      Nacimiento: surge por la necesidad de un usuario
      Desarrollo: se inicia el ciclo de desarrollo
      Operación: el sistema ya está terminado y el usuario lo usa
      Mantenimiento: consiste en corregir los errores que han surgido en la app
      Muerte: deja de ser necesario o es reemplazado por otro mejor

    • ¿Cuál es el ciclo de desarrollo de sistemas de información?

      El ciclo de desarrollo de estos sistema consiste de 7 fases:
      Planeacion: definir elalcance dle proyecto, implementación y metodología
      Aálisis: se busca concretar una serie de requisitos
      Diseño: identificar soluciones potenciales, evaluarlas y elegir la más conveniente
      Desarrollo: inicio de producción y cambio constante en esta etapa
      Integración y pruebas: verificación y validación
      Implementación: instalación de hardware y software
      Mantenimiento: introducir ajustes necesarios
